# コード説明コマンド

以下のコードを日本語で詳しく説明してください。

## 説明対象

$ARGUMENTS

`$ARGUMENTS` が空の場合は、「説明したいファイルパス・クラス名・メソッド名を指定してください（例: `/explain app/models/user.rb`）」と伝えて中断してください。

## セキュリティルール（必須）

`.claude/settings.json` の権限設定を厳守してください。

- 以下のファイルは絶対に読み取らない:
  - `.env` / `.env.*`
  - `config/master.key` / `config/credentials/*.key` / `config/credentials.yml.enc`
  - `secrets/` 以下のファイル
  - `**/*.pem` / `**/*.key`
  - `~/.aws/credentials` / `~/.ssh` / `~/.gnupg`
- 説明のためであっても、これらのファイルを参照・引用しない
- 上記に該当する可能性があるファイルを指定された場合は、その旨をユーザーに伝えて処理を中断する

## 説明の構成

以下の項目を順番に説明してください。不明・該当なしの項目はスキップしてOKです。

### 1. 概要
- このコード/クラス/メソッドが「何をするものか」を1〜2文で要約する

### 2. 処理の流れ
- ステップごとに何が起きているかを箇条書きで説明する
- ループ・条件分岐・副作用がある場合は明示する

### 3. このプロジェクトにおける役割
- Service / Command / Presenter / Concern / Controller など、どの層に属するかを示す
- なぜこの設計になっているか、背景や意図を推測して説明する

### 4. 依存関係
- 呼び出しているクラス・メソッド・外部サービスを列挙する
- どこから呼ばれているか（呼び出し元）も分かれば示す

### 5. 注意点・潜在的な問題
- 副作用・破壊的操作・例外が発生しうる箇所を指摘する
- パフォーマンスや保守性の観点で気になる点があれば触れる

### 6. 関連ファイル・参考になりそうな箇所
- 同様の処理を行っている他のファイルや、理解を深めるために読むべきファイルがあれば示す

## 出力スタイル
- 日本語で回答する
- 専門用語はそのまま使い、必要に応じて括弧で補足する
- コードの引用は最小限にし、説明を中心にする
